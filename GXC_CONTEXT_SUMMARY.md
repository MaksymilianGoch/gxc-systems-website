# GXC Systems — Compressed Project Context

## 1. Project Goal

GXC Systems is a website and automation project for local businesses, trades companies, service providers and SMEs in the DACH / Dreiländereck region.

The website must sell one clear operational system:

Website + Inquiry Widget + CRM Workflow + Automation + Follow-up + Reporting.

The visitor should understand within 3–5 seconds:

"GXC Systems turns website inquiries into structured leads, appointments and follow-ups."

The emotional reaction should be:

"Exactly. This is how my business should handle inquiries."

The website must not primarily communicate "we use AI".
It must communicate: "we build the system that captures, structures, follows up and converts inquiries."

---

## 2. Locked Design Direction

The locked design direction is:

Bright Operational System.

The website must feel:
- bright
- calm
- premium
- trustworthy
- product-demo focused
- clear for local businesses
- more like a real operational product than a generic AI agency site

Visual direction:
- warm off-white background
- white product surfaces
- graphite/black typography
- deep blue CTAs
- subtle gold/green status accents
- generous whitespace
- clean product UI
- usegago-like clarity
- TechnologyWire-inspired proof-first thinking

The site should feel like an operational desk or business cockpit for inquiries, appointments and follow-up.

---

## 3. Strategic Diagnosis

The site has improved structurally, but it still risks feeling like a generic AI/SaaS template.

The problem is not only color, spacing or layout.

The real missing piece is visual proof.

GXC currently explains the system, but it must show the system.

The page must visually prove how a customer inquiry becomes:
- captured
- structured
- entered into CRM
- assigned to the team
- followed up
- converted into an appointment or order

The goal is not to add more features.
The goal is to create clearer product proof.

## 5. No-Go Rules

Do not:
- redesign the full website without approval
- reinterpret the design direction
- return to dark AI/cyber aesthetics
- create generic SaaS card walls
- add random gradients or glow effects
- add random 3D
- add decorative effects without purpose
- use fake testimonials
- use unsupported hard claims
- use placeholder content
- change unrelated sections
- build everything at once
- create unused components
- edit files that are not actually rendered

Every change must be judged by:

"Did this make the website visibly better, or only different?"

If it only made the website different, it is not acceptable.
